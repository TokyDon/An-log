## The Only Real Blockers

### Blocker 1 — No `.env` file
Template is at `.env.example`. The file structure is:
```
EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
EXPO_PUBLIC_GEMINI_API_KEY=your-gemini-key
```

**How to get each:**
- **Supabase URL + Key**: supabase.com → New project → Settings → API → copy `URL` and `anon public` key
- **Gemini Key**: aistudio.google.com → Get API key → Create API key (free tier has 15 req/min)

### Blocker 2 — No Supabase project + migrations not run
Once you have a project, run these two in Supabase SQL editor (in order):
1. `supabase/migrations/001_initial_schema.sql`
2. `supabase/migrations/002_daily_scans.sql`

## Key Architecture Rules (Never Break These)

1. **Zustand stores: manual AsyncStorage ONLY** — never use `zustand/middleware/persist` (white page bug)
2. **Collection display: always merge** local `collectionStore.animons` + Supabase `animons` via `useMemo`, deduped by `id`
3. **GPS privacy**: region is city/country only — no precise coordinates stored ANYWHERE
4. **Gemini prompt returns**: `identified`, `commonName`, `ageStage`, `confidence` only — game stats generated from `speciesRegistry`, not from AI
5. **Branch rule**: never commit to `main` — all work on feature branches, PR to merge
